                 Импорт проводок из файла

 После входа в этот режим на экране появится запрос на ввод имени файла
из которого которого нужно осуществить импорт проводок.
В нижней части экрана располагается строка подсказки:


F1 помощь F2 загрузить F3 просмотр F4 очистить F5 настройка F6 выполнить F10 выход

F1  - получить на экран этот текст;
F2  - начать импорт проводок из указанного в меню файла; 
F3  - просмотреть на экране файл указанный в меню;
F4  - очистить строку ввода;
F5  - настроить список команд которые необходимо выполнить до начала импорта;
F6  - выполнить список введённых команд (F5);
F10 - выход.

 Если в командах файла настройки должно использоваться имя файла, то 
вместо имени файла нужно ввести ICEB_ZAMENA
Программа перед выполнением команд заменит эту цепочку символов на имя
файла.

	Этот режим позволяет загрузить проводки из файла. Формат записи
должен быть следующим:

0|1.1.2002|311|     |361|330|100.00|0.00|Платежка N23|           |Кирпичный завод|
|    |      |    |    |   |    |     |       |            |              |
|    |      |    |    |   |    |     |       |            |              ->Наименование контрагента счёта корреспондента.
|    |      |    |    |   |    |     |       |            |                Применяется для счетов с развёрнутым сальдо
|    |      |    |    |   |    |     |       |            -> Наименование контрагента счёта.
|    |      |    |    |   |    |     |       |               Применяется для счетов с развёрнутым сальдо
|    |      |    |    |   |    |     |       ->Комментарий (не обязательное поле для заполнения)
|    |      |    |    |   |    |     --> Кредит
|    |      |    |    |   |    --->Дебет
|    |      |    |    |   ---> Рекомендованный код контрагента для счета корреспондента. (не обязательое поле для заполнения)
|    |      |    |    |        Применяется для счетов с развёрнутым сальдо
|    |      |    |    --> Счет корреспондент
|    |      |    --> Рекомендованный код контрагента для счета. (не обязательное поле для заполнения)
|    |      |        Применяется для счетов с развёрнутым сальдо
|    |      ---> Счет
|    --> Дата проводки
---> 0- балансовая проводка, -1 - не балансовая проводка

	Поля "Код контрагента для счета", "Наименование контрагента счёта",
"Код контрагента для счета корреспондента", "Наименование контрагента счёта
корреспондента"
заполняются только для счетов с развернутым сальдо.
	Для проводок по небалансовым счетам поля "Счет корреспондент" и "Код
контрагента для счета корреспондента","Наименование контрагента счёта корреспондента" не заполняются.


        Если в проводке есть счёт с развёрнутым сальдо, то программа работает следующим образом:

Ищет по наименованию контрагента в справочнике контрагентов код
контрагента.

Если в справочнике есть контрагент с таким наименованием, то его код 
используется в проводке.

Если в справочнике нет контрагента с таким наименованием, то проверяет рекомендованный код контрагента.

Если рекомендованный код контрагента не указан, то программа записывает
наименование контрагента в справочник с уникальным кодом контрагента и
делает проводку с уникальным кодом контрагента.

Если рекомендованный код контрагента указан, то программа проверяет свободен
ли этот код в справочнике контрагентов.
Если в справочнике нет контрагента с таким кодом, то программа
добавляет в справочник наименование контрагента с рекомендованным кодом
и выполняет проводку с рекомендованным кодом контрагента.

Если в справочнике есть контрагент с таким кодом как рекомендованный код,
то программа записывает с справочник наименование контрагента и присваивает
ему уникальный код контрагента. После чего выполняет проводку с уникальным
кодом контрагента.

 При необходимости в этом файле можно указать реквизиты контрагента до 
записи с проводкой в которой этот контрагент будет использоваться.
Запись с реквизитами контрагента должна начинаться с символов:

KON|

В этой же строке должны быть перечислены все реквизиты контрагента
разделённые символом "|". Порядок реквизитов такой:

1  Рекомендованный код контрагента
2  Короткое наименование контрагента (единственный обязательный для заполнения реквизит)
3  Наименование банка
4  Адрес контрагента
5  Населённый пункт где находится банк
6  Код ЕГРПОУ
7  МФО
8  Номер счета
9  Индивидуальный налоговый номер
10 Номер свидетельства плательщика НДС
11 Телефон
12 Код группы контрагента (цифровой)
13 Налоговый адрес
14 Регистрационный номер частного предпринимателя
15 Полное наименование контрагента
16 Населённый пункт контрагента
17 0-общая система налогообложения 1-единый налог

Например:

KON|125|Мясокомбинат|Приватбанк|г. Киев улица Ленина 1/1|12345|2345|123456789|987654321|3452345|244545|0|||||0

 Обязателен для заполнения только реквизит номер 2 - Наименование контрагента.
Контрагент будет загружен в базу с кодом указанным в реквизите номер 1 только ели этот код не занят. Иначе 
программа сама присвоит ему свободный код.
